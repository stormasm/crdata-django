Analysis Best Practices

These are the best practices for using Analysis algorithms in CRdata.

Bar Plot   Create a bar plot with Vertical bars is best displayed with 20 or less rows in your data set.

Box Plot   Create box-and-whisker plots.

Heat Map   A graphical representation of data where the values are represented as colors is best displayed with 50 or less rows in your data set.

Multi Dimensional Scaling [MDS]   

Principal Component Analysis [PCA]   

Partial Least Squares Regression [PLSR]    Please use two data files when running PLSR analyses.

Pie Chart   Pie charts are best displayed with 25 or less rows in your data set.

Time Series   Plotting of x and y values are best displayed with 25 or less rows in your data set. When plotting time series please use the default Color Palette. The custom, user selected color palette's currently do not work.

Notes of Interest

  • When viewing the interactive plots of MDS, PCA, and PLS there is a checkbox called Enable Tooltip. When this box is checked it allows you to see the X and Y coordinates of the row data. The syntax of the X and Y coordinates is of the form "Rowname of X = Y".

  • The first row and column in CRdata input files are assumed to be row and column labels (names). These label names can be any mixture of the letters A - Z and numerals 0 - 9, otherwise known as alphanumeric characters.

  • When viewing the plots of MDS, PCA, and PLS there is a button called Scale Axis. This button allows one to set the X Min, X Max, Y Min and Y Max of the output graphs. If you use the Slider in connection with the Min and Max settings it allows one to both expand and contract the axes values along with limiting the scope of that scaling.
